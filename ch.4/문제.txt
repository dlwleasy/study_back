1. 선택 문제
1.1. 클라이언트와 서버 간의 관계에 대해 잘 설명한 것은 무엇인가요?

a) 클라이언트는 서버에게 요청을 보내고, 서버는 그 요청을 처리하여 클라이언트에게 결과를 반환한다.
b) 서버는 클라이언트에게 요청을 보내고, 클라이언트는 서버에게 결과를 반환한다.
c) 클라이언트와 서버는 서로 독립적으로 동작하며, 서로의 요청과 응답을 처리하지 않는다.
d) 서버는 클라이언트와의 연결을 유지할 필요가 없다.

1.2. 웹 브라우저가 웹 서버에 요청을 보내는 과정 중 DNS의 역할은 무엇인가요?

a) 웹 서버에서 요청을 처리하는 역할을 한다.
b) 도메인 이름을 IP 주소로 변환하여, 브라우저가 서버에 접근할 수 있도록 돕는다.
c) HTTP 요청을 서버로 전달하는 역할을 한다.
d) 클라이언트의 IP 주소를 변경하여 요청을 보내는 역할을 한다.

정답 
1. 선택 문제 답변
1.1. 클라이언트와 서버 간의 관계에 대해 잘 설명한 것은 무엇인가요?
답: a) 클라이언트는 서버에게 요청을 보내고, 서버는 그 요청을 처리하여 클라이언트에게 결과를 반환한다.

1.2. 웹 브라우저가 웹 서버에 요청을 보내는 과정 중 DNS의 역할은 무엇인가요?
답: b) 도메인 이름을 IP 주소로 변환하여, 브라우저가 서버에 접근할 수 있도록 돕는다.

2. 빈칸 채우기
2.1. 웹 서버는 클라이언트의 요청에 맞는 HTML 문서를 응답으로 반환한다.

2.2. IP 주소는 네트워크에서 장치를 구별하는 주소이며, 포트 번호는 애플리케이션을 구분하는 데 사용된다.

3. 설명 문제
3.1. 로드 밸런싱의 목적과 방법을 설명하시오.
답: 로드 밸런싱은 여러 클라이언트 요청을 여러 서버로 분산시켜서 서버에 과부하가 걸리지 않도록 도와준다. 이를 통해 서버가 효율적으로 자원을 사용할 수 있게 된다. 보통 스위치를 이용하여 클라이언트 요청을 분산시키며, 소프트웨어적으로 어떤 서버로 요청을 보낼지 결정할 수 있다.

3.2. 캐시의 목적과 그것이 웹 서버에서 어떻게 활용될 수 있는지 설명하시오.
답: 캐시는 비용이 큰 작업의 결과를 저장해 두고, 같은 요청이 들어올 경우 이미 저장된 데이터를 사용하여 처리 속도와 성능을 향상시킨다. 예를 들어, 자주 사용되는 데이터를 미리 저장해 두고 필요할 때 빠르게 반환함으로써 서버의 부하를 줄인다.

4. 서술형 문제
4.1. 브라우저가 웹 서버에 요청을 보내는 과정에서 IP 주소가 필요한 이유를 설명하시오.
답: 웹 브라우저는 요청을 보낼 때 IP 주소가 필요하다. IP 주소가 없으면 어떤 서버에 요청을 보낼지 알 수 없기 때문이다. DNS 서비스가 도메인 이름을 IP 주소로 변환해주며, 브라우저는 이 IP 주소를 이용해 알맞은 서버에 접근하게 된다.

4.2. 웹 브라우저와 웹 서버 간의 통신에서 "포트"가 필요한 이유를 설명하시오.
답: 포트는 애플리케이션을 구분하는 데 사용된다. 물리적으로는 IP 주소가 구분할 수 있지만, 같은 IP 주소 내에서 여러 애플리케이션이 실행되기 때문에 각 애플리케이션을 구분하기 위해 포트 번호가 필요하다. 예를 들어, 웹 서버는 보통 8080 포트를 통해 요청을 받는다.

5. 실습 문제
5.1. 다음 URL을 입력한 후, DNS가 어떻게 IP 주소로 변환되는지 설명하시오.
http://example.com:8080
답: URL에서 example.com은 도메인 이름이고, DNS는 이를 IP 주소로 변환해 웹 브라우저가 해당 서버에 접근할 수 있게 한다. 포트 번호 8080은 웹 애플리케이션이 요청을 받을 포트를 지정한다.

5.2. 클라이언트가 웹 서버로 HTTP 요청을 보내고 응답을 받는 과정에서 사용하는 포트 번호의 역할을 설명하시오.
답: 포트 번호는 동일한 IP 주소 내에서 여러 애플리케이션을 구분할 수 있도록 해준다. 예를 들어, 8080 포트는 웹 서버가 요청을 처리하는 포트로, 웹 서버 애플리케이션이 클라이언트의 요청을 받을 때 사용된다.
